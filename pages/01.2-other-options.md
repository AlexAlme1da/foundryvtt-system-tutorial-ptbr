# Outras opções para começar

Além de usar o Boilerplate System, você também pode usar o Simple World-building ou um dos outros sistemas boilerplate disponíveis, como flatpacktemplate.

## Começando com o sistema Simple World-building

Começar com o sistema Simple World-building (SWS) é uma boa opção se você quiser construir seus sistemas com base nos padrões mais recentes que Atropos construiu no FoundryVTT. O sistema é um dos dois sistemas oficiais para FoundryVTT, então você pode ter certeza de que o código lá será um ótimo exemplo de como construir as coisas do Jeito Certo™.

Se você usar o SWS como base, deve procurar nos arquivos por quaisquer instâncias de `worldbuilding` ou `simple` e renomeá-las para serem mais apropriadas para o nome do seu sistema.

Algo a se atentar é que o SWS usa atributos dinâmicos tanto para atores quanto para itens. Se seu sistema também não usar esses, você vai querer removê-los dos seus arquivos de template HTML e certificar-se de que também foram removidos do Javascript da ficha de ator e item.

## Começando com um boilerplate

Outra opção é usar um sistema boilerplate como sua base. Esses sistemas não são destinados a serem usados diretamente, mas são sistemas leves que incluem algumas coisas para você começar enquanto deixam a maior parte do trabalho para você.

Esta é a opção que eu recomendo para a maioria dos novos desenvolvedores, e também é a rota que este tutorial seguirá. Existem vários bons sistemas boilerplate disponíveis:

### flatpacktemplate

**URL**: [https://github.com/GateKept/flatpacktemplate](https://github.com/GateKept/flatpacktemplate)

O sistema flatpacktemplate é um conjunto inicial leve criado por GateKeeper do servidor Discord do Foundry. Ele inclui classes de item e ator de exemplo para você começar.

### Boilerplate System
**URL**: [https://gitlab.com/asacolips-projects/foundry-mods/boilerplate](https://gitlab.com/asacolips-projects/foundry-mods/boilerplate)

O Boilerplate System é um conjunto inicial leve que criei que inclui um pequeno número de exemplos (como pontuações de habilidade de personagem) e algumas classes CSS auxiliares para organizar suas fichas sem realmente ter que escrever novo CSS. Ele também inclui os arquivos fonte Sass que usei para compilar o CSS se você estiver confortável com NPM e Gulp.

Usaremos este sistema para o tutorial. Se você estiver confortável em usar NPM, também pode usar meu gerador Yeoman para gerar este sistema: [https://www.npmjs.com/package/generator-foundry](https://www.npmjs.com/package/generator-foundry). Depois de ter o gerador instalado, você só precisa executar `yo foundry` no diretório de sistemas e ele solicitará o nome do seu sistema antes de gerar os arquivos para você.

### Foundry Project Creator

**URL**: [https://www.npmjs.com/package/create-foundry-project](https://www.npmjs.com/package/create-foundry-project)

O Foundry Project Creator é definitivamente a opção avançada destes três, mas eu recomendo fortemente se você estiver procurando por um excelente CLI e gostaria de usar TypeScript para uma experiência de desenvolvedor mais robusta. Além de gerar sistemas, o Foundry Project Creator também permitirá que você gere módulos!

## Começando do zero

Você também pode começar completamente do zero! Esta é a opção mais avançada, pois você precisa já saber o que construir, mas mais informações podem ser encontradas em [https://foundryvtt.com/article/system-development/](https://foundryvtt.com/article/system-development/).
